2. DB도 아무거나 사용할 것(쿠폰저장 필요)
3. 비동기로 150명이 동시에 API콜을 요청함(aiohttp gather 메서드 활용할 것)
4. 요청 순서대로 150명 중에 100명은 쿠폰을 획득하고, 나머지 50명은 쿠폰획득 실패 처리를 하는 API를 만들 것
5. 쿠폰은 16자리의 임의의 문자열로 구성됨
6. 단, 100명은 모두 다른 쿠폰을 발급받아야 함
